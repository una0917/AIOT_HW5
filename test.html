Problem Definition

AI 偵測是什麼？— Binary Classification 視角

我們要解決的核心問題：「判斷一段文字是否由 AI 撰寫」。

數學化表示：

f(x) = P(AI ∣ x)

其中 x 為整段文本，我們希望設計一組特徵 φ(x)，讓 AI 生成文本與 Human 文本在特徵空間中可分離。

⚠️ 偵測 ≠ 鑑定：

  • 輸出的是「AI 機率」而非「司法鑑定」的絕對真相。
  • 適合作為輔助訊號、風險指標，而不是唯一決策來源。
（可放：AI vs Human 樣本文字，右側顯示偵測模型輸出的機率條）

Conceptual Contrast

AI 文本 vs 人類文本 — 本質差異

AI 生成文本的典型特徵：

  • 由大型語言模型根據條件機率產生 → 機率最優、連續穩定
  • 整體篇章結構與語氣 一致性高、邏輯流暢
  • 較少極端長句 / 斷裂，較少「情緒爆衝」式表達

Human 文本的典型特徵：

  • 跳躍性高、帶有噪音；情緒驟升驟降，容易跑題
  • 常見錯字、口語化、省略、未完成句
  • 主題可能出現 「topic drift」：突然轉向不相關話題

📊 視覺化建議：左右並排相同主題的 AI 文與 Human 文，標出句長、連接詞使用頻率與情緒用語分佈，作為後續指標（PP、Burstiness、Stylometry）的直覺示意。

Language Model Basics

語言模型如何看待一段文本？

語言模型會為每一個 token（詞或子詞）計算條件機率：

q(wᵢ ∣ w₁, …, wᵢ₋₁)

整段文本的 likelihood 可以寫成：

q(w₁, …, wₙ) = ∏ᵢ q(wᵢ ∣ context)
  • · AI 文本：模型自己生成 → 通常位於高機率區域，likelihood 較高。
  • · Human 文本：有時會使用模型不常見的組合 → likelihood 較低且波動大。
（可放：簡化的 Transformer / BERT 方塊圖，右側輸出為 token 機率分佈條）

Perplexity · 基礎公式

交叉熵與困惑度（Perplexity）

交叉熵 H：

H = - (1 / N) · Σ log q(wᵢ)

困惑度 PP：

PP = exp(H)

直覺解讀：

  • PP 越低 → 文本對模型而言越「可預期」，貼近高機率語言。
  • PP 越高 → 模型越「困惑」，代表文本包含不尋常用字 / 結構。
  • 我們不只看 平均 PP，更關注 PP 的時間序列 與波動。

Perplexity · 應用

為什麼 Perplexity 對 AI 偵測有用？

Human vs AI PP 型態：

  • AI 文本：PP 通常 較低且平滑，波動幅度小。
  • Human 文本：PP 波動較大，常出現局部高峰（spikes）。

背後原因：

  • 人類會犯錯、情緒化、突然換說法 → 出現模型較少見的 token 組合。
  • AI 經訓練後習慣待在「高機率語言區」，避免太多驚喜。

Sentence Rhythm · Burstiness

句子節奏統計：Burstiness 指標

定義：

Burstiness B = σ / μ

其中 μ 為平均句長，σ 為句長標準差。

直覺解釋：

  • AI：句長穩定，B 值偏低 → 中庸句長的規則節奏。
  • Human：長短句交錯，B 值偏高 → 時而長句爆發、時而短句切斷。

Burstiness · 應用

為什麼 Burstiness 能幫忙抓出 AI？

語言模型生成句子的統計傾向：

  • 為保持可讀性與流暢度，模型偏好落在「合理句長」區間。
  • 極短句與極長句比例被壓低，節奏呈現「可預期的中庸」。

人類寫作的爆發性：

  • 情緒上來時會一口氣寫很長：「這也太扯了我真的是……」。
  • 也會突然只打一句：「太扯了。」

Burstiness · Distribution

句長分布：單峰 vs 多峰

觀察句長的機率分布，可以看見 AI 與 Human 在節奏上的結構差異。

  • AI：句長分布通常呈現單峰、尾巴短。
  • Human：分布多峰，且長尾拉得較長。
（圖示 placeholder：AI / Human 句長 histogram 對照）

Stylometry · 風格指紋

Stylometry：把寫作習慣變成特徵

Stylometry 原本用在「作者鑑定」領域，核心思想是： 把寫作風格當成一種可量化的指紋。

  • 從用字、句法、情緒與噪音層面，抽取一組高維特徵。
  • 同一作者在這些特徵空間中會形成穩定的「風格叢聚」。
  • AI 與 Human 也可以被視為兩個「作者族群」。

Stylometry · Lexical Layer

用字特徵：Lexical 指標

  • TTR（Type-Token Ratio）詞彙多樣性
  • Function words（例如「而且」、「因此」）比例
  • 生僻詞與專有名詞的出現頻率
  • 人類錯字率與不規則縮寫

AI / Human 差異：

  • AI：Function words 模板化、句首句尾結構高度重複。
  • Human：常出現 idiosyncratic 用字、個人慣用詞、打字錯誤。

Stylometry · Syntactic Layer

句法特徵：POS / Syntax 結構

常見句法特徵：

  • POS tag 分布與 POS n-gram
  • 句法樹深度、從句結構與修飾語位置
  • 被動語態、條列句、條件句比例

AI 的句法足跡：

  • 句型相對工整、結構重複度高。
  • 從句分布更可預測，較少半途而廢的句子。

Stylometry · Emotion & Noise

情緒與雜訊層：人類的「不完美」

Human 文本特徵：

  • 情緒曲線起伏明顯：越寫越激動或越寫越冷靜。
  • 錯字、emoji、口語、語助詞（啊、欸、真的假的）。
  • 打字習慣（少空格、多逗點、不對稱括號）。

AI 文本特徵：

  • 情緒多被平滑化，較少極端用詞。
  • 幾乎沒有拼字錯誤，標點規則化。
  • 用字偏書面、中性、禮貌。

Advanced Linguistics · Zipf's Law

Zipf’s Law：自然語言的長尾分布

基本定律：

f(r) ∝ 1 / r

排名越後面的詞，出現頻率越低；但自然語言中尾端仍有相當長的一段「長尾」。

AI 的問題：

  • 若訓練或解碼策略過度保守，模型會少用長尾詞。
  • 缺乏罕見但極具人味的字詞 → 文字看起來「太正常」。

Zipf's Law · Visualization

log–log Zipf 曲線：尾巴的故事

將詞彙頻率畫在 log(rank)–log(freq) 座標上，人類語料通常呈現較長的尾巴。

  • Human：曲線尾巴較平，表示長尾詞維持一定比例。
  • AI：若過度集中主流詞，尾巴會變短、曲線較陡。

Semantic Space · Drift

語意軌跡：Embedding Drift

定義：

Driftₜ = ∥ eₜ − eₜ₋₁ ∥

eₜ 為第 t 個 token 的 embedding，Drift 代表語意在相鄰 token 間的位移量。

理論觀察：

  • AI：語意走向較穩定，Drift 分布偏小。
  • Human：容易突然轉彎、插入側題 → 出現較大的 Drift。

Semantic Space · Variance

語意變異度：Variance / Kurtosis / Coherence

做法：

  • 取整段文字的所有 token embeddings。
  • 計算 variance、kurtosis 與 coherence score。

觀察指標：

  • AI：語意向量聚集較緊密，一致度高。
  • Human：變異度較大，甚至會形成多個不同主題的叢聚。

Semantic Space · Trajectory

語意軌跡 2D 投影

將 token embedding 透過 PCA / t-SNE 投影到 2D 平面，並依照時間連線，即可看到文本的語意散步路徑。

  • AI：路徑平順、折點少。
  • Human：路徑跳躍，多次大幅轉向。

XAI · What the Model Sees

可解釋 AI：偵測器到底在抓什麼特徵？

AI 文本高相關特徵：

  • 模板化連接詞：「因此」、「總結來說」、「值得注意的是」。
  • 過度一致的開場與結尾句型。
  • 中性而禮貌的語氣，極少粗俗或強烈情緒詞。

Human 文本常見高權重特徵：

  • 語氣詞與口語片語：「欸」、「超扯」、「真的假的」。
  • 錯字、斷句、表情符號。
  • topic jump：突然轉向個人經驗或八卦。

XAI · Token Attribution

SHAP / Integrated Gradients 熱力圖

透過 SHAP 或 IG，我們可以標示出「哪些 token 對 AI / Human 分類影響最大」，讓偵測器不再是黑箱。

  • AI：連接詞、模板句 opening / closing 常亮起高權重。
  • Human：稀有詞、噪音 token、情緒化詞彙常成為關鍵指標。

Detection Model · Classical ML

傳統機器學習：特徵向量 + Logistic Regression

特徵向量設計：

φ(x) = [ PP, B, TTR, POS, Zipf-tail, Drift, … ]

將前面所有統計與語意指標組合成一個高維向量。

Logistic Regression 決策：

P(AI ∣ x) = σ(wᵀ φ(x) + b)

優點：可解釋、計算量低、容易部署；缺點：難以捕捉複雜語意與長距依賴。

Detection Model · Deep Learning

BERT-based 偵測：CLS 向量 + Sigmoid

架構：

P(AI ∣ x) = σ(W · CLS(x) + b)

其中 CLS(x) 為 BERT 對整段文本的聚合表示向量。

Hybrid Model：

  • 可將 CLS 向量與人工特徵 φ(x) 拼接。
  • 結合「深度語意」與「可解釋統計指標」，提升偵測效果與說服力。

Operational Design · Threshold

偵測閾值（Threshold）如何設定？

高閾值的策略：

  • 只有非常高機率時才標記為 AI。
  • 優點：保護人，避免錯誤指控 Human。
  • 適用：學術倫理、申請文件、敏感場景。

低閾值的策略：

  • 寧可多抓，也接受一定 False Positive。
  • 適用：Spam / Bot 偵測、內容審查預警。

Limitations · Real World

實務上的限制與挑戰

  • 新一代 LLM（特別是指令微調與人類編輯後文本）越來越像人類。
  • 混合文本：AI 生成初稿，人類改寫後，風格變得模糊。
  • Domain shift：科學論文、社群貼文、程式碼等語域差異巨大。
  • 多語言情境：不同語言需重新標定特徵分布與閾值。

Adversarial Attack · Typo Injection

對抗攻擊（一）：刻意插入錯字與噪音

策略：

  • 先由 AI 生成文章，再刻意插入錯字、亂空格與口語片語。
  • 目標是讓 Perplexity ↑、Burstiness ↑，模仿人類噪音。

影響：

  • 若偵測器只依賴表層統計指標，可能被誤導。
  • 因此需要結合語意空間與 Stylometry 深層特徵。

Adversarial Attack · Sampling

對抗攻擊（二）：調整生成參數增加「人味」

常見手法：

  • 調高 Temperature。
  • 增大 top-k / top-p。
  • 刻意加入隨機改寫、同義詞替換。

結果：

  • 使 Zipf 尾巴變長、句長分布更接近 Human。
  • 迫使偵測器必須依賴更「深層」的語意與風格特徵。

System Overview · Detection Pipeline

完整 AI 偵測架構：從特徵到決策

五大模組：

  • 語言模型特徵：PP、log prob variance。
  • 句子節奏：Burstiness、句長分布。
  • Stylometry：Lexical / Syntax / Emotion / Noise。
  • 語意空間：Drift、Variance、Coherence。
  • 分類器：傳統 ML / BERT / Hybrid。

可對應到實作流程：

  • 前處理 → 特徵抽取 → 模型推論 → XAI 解釋 → 報表輸出。
  • 可進一步包裝成 Web 介面，供老師 / 管理者直接貼上文字查看 AI 機率。

Conclusion

結語：AI 偵測是一門跨學科的科學

AI vs Human 的差異，可以透過多個維度被量化：

  • 統計語言學指標：Perplexity、Burstiness、Zipf’s Law。
  • 高階語言學與風格學：Stylometry、多層風格特徵。
  • 語意向量空間：Embedding Drift / Variance / Coherence。
  • 深度學習模型與 XAI 解釋工具。

關鍵提醒：

偵測 ≠ 鑑定 — 我們提供的是「訊號與機率」，協助人類做更負責任的判斷，而不是替代司法或倫理決策。
